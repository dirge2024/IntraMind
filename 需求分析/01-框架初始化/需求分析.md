# 框架初始化 — 需求分析

## 核心目标
建立多模块 Maven 工程骨架，保障模块间依赖单向流动，验证基础设施可连通。

## 关键决策

| 决策项 | 约束 | 风险 |
|--------|------|------|
| Spring Boot 版本 | 3.x，兼容 Java 17 | 选 3.3.x 稳定版，避免最新版踩坑 |
| Maven 父子结构 | 父 POM 统一版本管理 | dependencyManagement 和 modules 顺序决定构建顺序 |
| 模块依赖方向 | 单向传递，禁止反向 | 编译期检查 |
| 双实现抽象 | embedding/llm 接口化 | 切换实现不改调用方 |

## 核心模块（初始化阶段）

### 父 POM
- dependencyManagement 统管所有依赖版本号
- modules 声明，顺序与依赖方向一致
- Spring Boot Starter Parent 作为父 POM
- 关键 Starter：spring-boot-starter-web, spring-kafka, elasticsearch-java

### common 模块
- Result<T> 统一返回体（code/message/data）
- BaseException + GlobalExceptionHandler
- 不引入重依赖

### 子模块模板
- 每个模块含 pom.xml + src/main/java/com/localrag/{module}/ + src/test/java/
- 每层必须有 interface 包和 impl 包

### docker-compose.yml
- MinIO: 9000(API) / 9001(Console)，自动创建 bucket
- Kafka: KRaft 模式，无 Zookeeper
- Elasticsearch: 单节点，禁用 Security

### api 骨架
- 主启动类 + 健康检查 /api/health
- 全局 CORS + 异常处理注册

## 边界条件

| 场景 | 处理 |
|------|------|
| 模块间循环依赖 | Maven 编译期报错，common 不引用业务模块 |
| common 膨胀 | 只有被≥2个模块共用的类才进 common |
| 配置文件冲突 | 每模块只写自己的配置，api 用 spring.config.import 合并 |
| 敏感信息泄露 | 密码/Key 用 ${ENV_VAR} 占位，提供 .env.example |
| ES 客户端冲突 | 排除 spring-boot-starter-data-elasticsearch，用 elasticsearch-java |
| Kafka group-id | 预留配置项，避免后期硬编码 |

## 技术难点

1. **Maven 版本仲裁** — MinIO/ES/Kafka 客户端版本可能和 Boot BOM 不一致，需显式声明
2. **ES 客户端选型** — 不用 Spring Data ES，直接用 elasticsearch-java 8.x
3. **Bean 扫描** — api 模块需 @ComponentScan(basePackages = "com.localrag")
4. **SSE 预留** — api 模块预留 SseEmitter，不实现
5. **Kafka KRaft** — KAFKA_CFG_PROCESS_ROLES=broker,controller 必须配对

## 验收标准

| 编号 | 输入 | 输出 | 标准 |
|------|------|------|------|
| 1 | 无 | docker-compose.yml | 三个中间件可健康访问 |
| 2 | 无 | 父 pom.xml | mvn clean compile 全模块通过 |
| 3 | 无 | common 模块 | Result/Exception 编译通过 |
| 4 | 无 | api 模块 | /api/health 返回 {"code":200,"data":"ok"} |
| 5 | 无 | .env.example | 列出所有环境变量及含义 |

## 执行步骤

```
1. docker-compose.yml
2. 父 pom.xml（含 dependencyManagement）
3. common 模块（pom.xml + Result + Exception）
4. 其余 7 个子模块的空骨架
5. api 模块（启动类 + health 接口）
6. .env.example + .gitignore
7. mvn clean compile 验证
8. 启动 api，验证 health 接口
```
