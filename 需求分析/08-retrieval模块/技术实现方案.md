# Retrieval 模块 — 技术实现方案

## 1. 依赖

```xml
<dependency>com.localrag:common</dependency>
<dependency>com.localrag:embedding</dependency>
<dependency>com.localrag:vector-store</dependency>
<dependency>org.projectlombok:lombok</dependency>
```

ES 客户端已在 vector-store 模块创建，retrieval 通过依赖 `vector-store` 获取 `ElasticsearchClient` bean。

## 2. 接口设计

```
retrieval/contract/
└── RetrievalService
    └── List<RetrievalResult> search(String query, int topK)

retrieval/impl/
└── RetrievalServiceImpl  # ES KNN + BM25 + rescore
    ├── 依赖 EmbeddingService（query 向量化）
    └── 依赖 ElasticsearchClient（ES 查询）

retrieval/model/
└── RetrievalResult      # {chunkId, md5, text, charCount, score}
```

## 3. 三级瀑布检索

```java
public List<RetrievalResult> search(String query, int topK) {
    // 1. query → vector
    float[] vector = embeddingService.embed(List.of(query)).get(0);

    // 第一轮：KNN 向量召回 Top 30
    var knnResults = client.search(s -> s
        .index("localrag-chunks")
        .knn(k -> k
            .field("denseVector")
            .queryVector(toList(vector))
            .k(30)
            .numCandidates(100)
        )
    , Map.class);

    List<String> knnIds = knnResults.hits().hits().stream()
        .map(h -> (String) ((Map) h.source()).get("chunkId"))
        .toList();

    if (knnIds.isEmpty()) return List.of();

    // 第二轮：在 KNN 的 30 条上 BM25 重排
    var bm25Results = client.search(s -> s
        .index("localrag-chunks")
        .query(q -> q.bool(b -> b
            .must(m -> m.ids(i -> i.values(knnIds)))  // 限定在 30 条内
            .must(m -> m.match(mm -> mm.field("text").query(query)))
        ))
        .size(15)  // BM25 得分 Top 15 进入精排
    , Map.class);

    // 第三轮：rescore 精排 → Top K
    var finalResults = client.search(s -> s
        .index("localrag-chunks")
        .query(q -> q.ids(i -> i.values( // 在 BM25 的 15 条内精排
            bm25Results.hits().hits().stream()
                .map(h -> (String) ((Map) h.source()).get("chunkId"))
                .toList()
        )))
        .rescore(r -> r
            .windowSize(15)
            .query(rq -> rq
                .rescoreQuery(rqq -> rqq.match(m -> m
                    .field("text").query(query)))
                .queryWeight(0.2)
                .rescoreQueryWeight(0.8)
            ))
        .size(topK)
    , Map.class);

    return mapResults(finalResults);
}
```

## 4. 瀑布策略说明

```
KNN(30) → BM25 in [30] → Top 15 → rescore in [15] → Top 10
```

每步缩小候选集：30 → 15 → 10。BM25 只在 KNN 召回的小范围内重排，避免全局 BM25 的嘈杂干扰。

## 5. API 端点

```
POST /api/retrieval/search
{
  "query": "什么是 RAG？",
  "topK": 10
}

Response:
{
  "code": 200,
  "data": [
    {"chunkId": "...", "text": "...", "score": 0.943},
    {"chunkId": "...", "text": "...", "score": 0.871}
  ]
}
```

## 6. RetrievalResult 模型

```java
@Data @Builder
public class RetrievalResult {
    private String chunkId;
    private String md5;
    private String text;
    private int charCount;
    private double score;
}
```
