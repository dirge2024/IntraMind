# Messaging 模块 — 需求分析

## 核心业务逻辑

文件上传完成后，storage 模块通过 Kafka 发送消息，通知 document 模块进行文档解析和分块。messaging 模块提供 Kafka 生产者和消费者的基础封装，隔离各模块对 Kafka API 的直接依赖。

### 消息流

```
storage (上传完成)
  │
  │  KafkaMessage<FileUploadedPayload>
  │  topic: document.uploaded
  │
  ▼
document (消费消息 → 解析分块)
  │
  │  KafkaMessage<DocumentChunkedPayload>
  │  topic: document.chunked
  │
  ▼
embedding (消费消息 → 向量化)
  │
  │  KafkaMessage<EmbeddingRequestedPayload>
  │  topic: embedding.requested
  │
  ▼
vector-store (消费消息 → 存入 ES)
```

### 设计原则

- **生产者和消费者分离**：发消息的模块调 `MessageProducer`，收消息的模块实现 `MessageConsumer`
- **消息体标准化**：统一包装 `{messageId, type, payload, timestamp}`，payload 用泛型
- **序列化用 JSON**：简单直接，前期够用，后期可切 Avro
- **发送即忘 + 异常重试**：发消息失败不阻塞主流程，只打日志

## 关键功能模块

### 1. 消息生产者（MessageProducer）

- 接口：`send(String topic, T payload)`
- 实现：Spring KafkaTemplate 包装
- 发送失败回调：`ListenableFutureCallback` 打 error 日志
- 不抛异常阻塞调用方

### 2. 消息消费者（MessageConsumer）

- 抽象类：定义 `onMessage(T payload)` 模板方法
- 子类只需实现业务逻辑，反序列化和异常处理由框架层负责
- 消费失败：记录 offset，不自动提交，人工介入

### 3. 消息体定义

- `KafkaMessage<T>`：`{messageId, topic, payload, timestamp}`
- Payload DTO：`FileUploadedPayload`（本次只定义这一个，后续模块各自扩展）

### 4. Kafka 配置

- 生产者：JSON 序列化，acks=1（leader 确认即可），batch 发送
- 消费者：JSON 反序列化，group-id = `localrag-{module}`，手动提交
- 连接配置从 `application.yml` 读

## 边界条件

| 场景 | 处理策略 |
|------|----------|
| Kafka 不可用 | 生产者异步回调打 error 日志，不阻塞上传成功返回 |
| 消息发送失败 | 不重试（Kafka 层面有 batch retry），依赖后续补偿机制 |
| 消费者处理失败 | 打 error 日志，offset 不提交，消息不丢失 |
| 重复消费 | 消费者通过 messageId 幂等判断（暂不做，后续加） |
| Topic 不存在 | Docker 启动时已预建 3 个 topic，无需代码创建 |
| 消息体版本兼容 | JSON 序列化，字段新增用 `@JsonIgnoreProperties(ignoreUnknown=true)` |

## 潜在技术难点

**1. 多模块共享消息 DTO（难度：低）**

`FileUploadedPayload` 定义在 messaging 模块，storage 发消息用，document 收消息也用。messaging 作为公共依赖被两者引入，符合当前依赖方向（storage → messaging ← document 不成立，需调整）。

> 解决：document 依赖 messaging 即可。数据流是 storage → messaging，document → messaging。storage 不直接依赖 messaging 时，DTO 放 common 更合理。详见技术方案。

**2. 消费者 group-id 隔离（难度：低）**

多个消费者消费同一 topic 需要不同 group-id，否则只有一个能收到消息。规范：`localrag-{模块名}`。

**3. Spring Kafka 自动配置（难度：低）**

Spring Boot 自动配置 Kafka，只需配 `spring.kafka.bootstrap-servers`。默认序列化器需改为 JSON。

## 输入输出规范

### 发送消息（由 storage 在 upload complete 后调用）

```java
FileUploadedPayload payload = FileUploadedPayload.builder()
    .md5("d41d8cd...")
    .fileName("项目文档.pdf")
    .fileSize(104857600)
    .objectKey("d41d8cd.../项目文档.pdf")
    .build();

messageProducer.send("document.uploaded", payload);
```

### 消息体 JSON 格式

```json
{
  "messageId": "uuid",
  "topic": "document.uploaded",
  "timestamp": "2026-05-05T17:30:00",
  "payload": {
    "md5": "d41d8cd...",
    "fileName": "项目文档.pdf",
    "fileSize": 104857600,
    "objectKey": "d41d8cd.../项目文档.pdf"
  }
}
```
