## 3. DocumentConsumer 顺序执行链

```java
@KafkaListener(topics = "document.uploaded", groupId = "localrag-document")
public void onMessage(String json) {
    KafkaMessage<FileUploadedPayload> msg = deserialize(json);
    FileUploadedPayload payload = msg.getPayload();

    // 1. 去重
    if (processed.contains(payload.getMd5())) return;

    // 2. 下载
    String url = storageService.getPresignedUrl(bucket, payload.getObjectKey(), 60);
    InputStream stream = download(url);

    // 3. Tika 解析
    String text = documentParser.parse(stream);

    // 4. 分块
    List<Chunk> chunks = textChunker.chunk(text, 1500, 150);

    // 5. 发送消息
    messageProducer.send("document.chunked", DocumentChunkedPayload.builder()
        .md5(payload.getMd5())
        .fileName(payload.getFileName())
        .chunks(chunks)
        .build());

    // 6. 标记已处理
    processed.add(payload.getMd5());
}
```

每步依赖上一步结果，同步顺序执行。异常时不提交 offset。

## 4. TikaDocumentParser

```java
@Component
public class TikaDocumentParser implements DocumentParser {
    private final Tika tika = new Tika();

    @Override
    public String parse(InputStream stream) {
        return tika.parseToString(stream);
    }
}
```

Tika 自动检测格式、处理编码，无需关心文件扩展名。

## 5. TextChunker 算法（流式 + 三级降级）

```java
public interface TextChunker {
    /**
     * @param text       输入文本（Tika 解析结果）
     * @param bufferSize 触发分块的缓冲区大小（5000）
     * @param minChunk   最小块大小（512）
     * @param overlapPct 重叠百分比（0.15）
     * @return 分块列表
     */
    List<Chunk> chunk(String text, int bufferSize, int minChunk, double overlapPct);
}
```

**算法流程**：

```
buffer 累积到 5000 字符后处理 →

1. 按 \n\n 分割段落
2. 遍历每个段落：
   ├─ 段落长度 < 512 → 累积到 currentChunk
   │     └─ currentChunk 累积到 >= 256 → 输出一个块
   ├─ 段落长度 >= 512 → 按句子分割（[。！？；\n]）
   │     └─ 句子长度 < 512 → 独立输出为一个块
   │     └─ 句子长度 >= 512 → HanLP 分词 → 按 512 token 切分
   │           └─ 超长句切分后各块仍不足 → 强制 512 字符截断
3. 相邻块之间：tail = chunk_i.substring(chunk_i.len * 0.85)
               chunk_i+1 = tail + 下一段内容
4. 剩余 buffer 不足 5000 → 直接按段落输出
```

**HanLP 使用**：只在超长句子时调用，全部分词后按 512 字符切分，保证块大小均匀。

## 6. ES 向量存储（vector-store 模块，后续）

每个 Chunk 存入 ES 的字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| chunkId | keyword | 唯一标识 |
| md5 | keyword | 来源文件 |
| text | text | 分块文本（用于 BM25） |
| charCount | integer | 字符数 |
| denseVector | dense_vector(2048) | 向量（embedding 模块生成） |
| index | integer | 块序号 |

**检索流程**（retrieval 模块，后续）：

```
用户查询 → embedding 向量化(2048) → ES KNN(denseVector, topK=10)
  → BM25 重排序(text 字段) → 返回 top5
```

KNN 语义召回 + BM25 关键词精排，两者互补。
