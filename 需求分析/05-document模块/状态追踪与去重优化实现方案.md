# 文件状态追踪 + 去重逻辑优化 — 实现方案

## 1. FileMetadata 状态枚举

```java
public enum Status {
    UPLOADING, READY, CHUNKED, EMBEDDED, DELETED
}
```

## 2. 各节点状态更新

### StorageController — UPLOADING → READY

```
initUpload()     → MySQL INSERT status=UPLOADING（预留，当前跳过）
completeUpload() → MySQL UPDATE status=READY
directUpload()   → MySQL INSERT status=READY
```

### DocumentConsumer — READY → CHUNKED

```java
// 分块 + 发送 document.chunked 后
fileMetadataRepository.findByMd5(md5);
metadata.setStatus(FileMetadata.Status.CHUNKED);
fileMetadataRepository.save(metadata);
```

### VectorStoreConsumer — CHUNKED → EMBEDDED

```java
// ES 写入成功后
fileMetadataRepository.findByMd5(md5);
metadata.setStatus(FileMetadata.Status.EMBEDDED);
fileMetadataRepository.save(metadata);
```

vector-store/pom.xml 加 `com.localrag:storage` 依赖。

## 3. 去重规则调整

```java
// initUpload
FileMetadata existing = fileMetadataRepository.findByMd5(md5);
if (existing != null && existing.getStatus() != FileMetadata.Status.DELETED) {
    if (existing.getStatus() == FileMetadata.Status.EMBEDDED) {
        // 全链路已完成 → 秒传
        return EXIST;
    }
    // 状态 < EMBEDDED + 同名 → 检查是否需要续传
    // 不同名 → 覆盖
}
```

## 4. 删除逻辑

```java
@DeleteMapping("/file/{md5}")
public Result<Void> deleteFile(@PathVariable String md5) {
    FileMetadata meta = fileMetadataRepository.findByMd5(md5);
    if (meta != null) {
        storageService.removeObject(meta.getBucket(), meta.getObjectKey());
        meta.setStatus(FileMetadata.Status.DELETED);
        fileMetadataRepository.save(meta);
    }
    return Result.ok();
}
```
