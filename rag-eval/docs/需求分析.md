## 二、500 条查询分配

### 类型与数量

| 查询类型 | 数量 | 正例数/条 | 构造思路 |
|---------|:---:|:--------:|---------|
| 精确术语 | 100 | 1-2 | 从文档中取具体命令、参数、版本号，直接作为 query |
| 描述/解释 | 120 | 2-4 | 概念定义，跨文档可多正例 |
| 数值/列表 | 70 | 1-3 | 文档中的枚举、步骤、分类 |
| 是否判断 | 60 | 1-2 | 将知识点改写为判断题 |
| 对比/差异 | 70 | 3-5 | **跨文档对比**，正例最多 |
| 长尾/模糊 | 50 | 1-3 | 口语化、省略、同义词替换 |
| 知识库不存在 | 30 | 0 | 问文档库没覆盖的领域 |
| **合计** | **500** | | |

## 三、标注方法

### 标准：正例 = 包含完整答案的 chunk

```
query: "Docker 怎么构建镜像"
正例 chunk（Docker文档）:
  c_docker_055: "使用 docker build 命令可以构建镜像，需要编写 Dockerfile..."
  c_docker_062: "Dockerfile 常用指令包括 FROM、RUN、COPY、CMD..."
负例 chunk（同文档但无关）:
  c_docker_010: "Docker Compose 用于定义多容器应用..."
  c_docker_140: "Docker Swarm 是 Docker 原生的集群管理工具..."
```

### 标注记录格式

```
queries.json 一行:
{
  "query_id": "q001",
  "query": "Docker 怎么构建镜像",
  "type": "描述/解释",
  "positive_pids": ["c_docker_055", "c_docker_062"],
  "negative_pids": ["c_docker_010", "c_docker_140"]
}
```

## 四、执行流程

```
1. 从 ES 导出所有 chunk → chunks.csv (chunkId, md5, fileName, text前100字)
2. 逐文档阅读 chunk 摘要，手工构造 query + 标注正例
3. 填写 Excel → 转 queries.json
4. 直接复用现有 scripts/evaluate.py 跑评测
5. 跑 scripts/report.py 出报告
```

## 五、预期结论

| 查询类型 | 预期混合优于 BM25？ | 原因 |
|---------|:---:|------|
| 精确术语 | 持平或略优 | 关键词已足够 |
| 描述/解释 | 显著优于 | 同义表达多 |
| 对比/差异 | 显著优于 | 跨文档语义关联 |
| 长尾/模糊 | 大幅优于 | 纯靠语义兜底 |
| 知识库不存在 | 无返回 | 不胡编 |
