## 二、为什么这 8 份够

8 份文档覆盖了测试多正例检索所需的全部条件：

| 测试场景 | 如何体现 | 示例 |
|---------|---------|------|
| 精确术语 | 同文档内搜配置参数、命令 | "Redis 的 RDB 持久化配置参数叫什么" |
| 语义相同/关键词不同 | 跨文档搜同一概念 | "容器化部署" vs "Docker 入门" = 同一个回答在 Docker 文档 |
| 多正例 | 一个概念多篇文档都有 | "索引优化" → Redis 索引 + ES 索引 + MySQL 索引都有 |
| 知识库不存在 | 问文档库没覆盖的 | "MongoDB 怎么配置副本集"(没上传MongoDB文档) |
| 跨文档归并 | 需要综合多篇 | "关系型和非关系型数据库的对比" → MySQL + Redis |
| 口语/模糊 | 口语化提问 | "那个缓存的东西咋配置" → 对应 Redis |

## 三、标注方法（500 条 query 的构建）

### 步骤 1：上传文档并入库

```bash
# 上传 8 份 PDF → 等待 ES 状态变为 EMBEDDED
# 确认每份文档的分块数（约 50-200 chunks/份）
```

### 步骤 2：导出所有 chunk 信息

从 ES 中导出所有 chunk 列表（chunkId + text + md5 + 文件名），方便标注时对照。

### 步骤 3：按 6 种类型分配 query 数量

| 类型 | 数量 | 构造方法 |
|------|:---:|---------|
| 精确术语 | 120 | 从文档中找具体配置参数、命令、版本号，直接作为 query |
| 描述/解释 | 150 | 从文档中找概念定义、原理说明段落，改写为问句 |
| 数值/列表 | 80 | 找文档中的枚举列表（5种类型、3个步骤），改写为"有哪些" |
| 是否判断 | 70 | 找有明确对错的陈述，改写为"xxx对吗"、"能不能xxx" |
| 对比/差异 | 50 | 找文档中的对比表格、对比段落，改写为"A和B的区别" |
| 长尾/模糊 | 30 | 故意用口语化/省略/错别字表述一个文档里的概念 |

### 步骤 4：标注正例（关键步骤）

对每条 query，在 chunk 列表中找到**所有**包含答案的 chunk 作为正例：

```
query: "Redis 有哪几种持久化方式？"
正例 chunk（可能跨文档）：
  - Redis文档 chunk_042: "Redis提供了两种持久化方式：RDB和AOF..."
  - Redis文档 chunk_045: "RDB持久化通过快照实现..."
  - Redis文档 chunk_046: "AOF持久化记录每次写操作..."
  - 数据库对比文档 chunk_018: "与MySQL的持久化不同，Redis采用..."

标注结果: positive_pids = [chunk_042, chunk_045, chunk_046, chunk_018]
```

**避免假负例**：标注时要把**所有相关 chunk**都标出来，不只是最精准的那一条。否则模型召回到了但标注说没召回，会压低真实 Recall。

### 步骤 5：标注知识库不存在的 query

```
类型: "知识库不存在"
数量: 30 条（从"是否判断"和"描述/解释"中各减 30 条挪过来）

构造方法: 问文档库没有的主题
  - "MongoDB 副本集怎么搭建"（没传MongoDB文档）
  - "React Hooks 有哪些"（没传前端文档）

预期: positive_pids = []（空列表）
考量: 系统是否返回空结果或说"暂无相关信息"，而不胡编
```
