## 3. 实现策略

### 3.1 JWT 生成与验证

```java
// JwtUtils.java
@Component
public class JwtUtils {

    private final SecretKey key;

    public JwtUtils(@Value("${localrag.auth.jwt-secret-key}") String base64Key) {
        byte[] keyBytes = Base64.getDecoder().decode(base64Key);
        this.key = new SecretKeySpec(keyBytes, "HmacSHA256");
    }

    public String generate(Long userId, String username, String role) {
        return Jwts.builder()
                .subject(String.valueOf(userId))
                .claim("username", username)
                .claim("role", role)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + 7 * 86400_000L))
                .signWith(key)
                .compact();
    }

    public JwtPayload validate(String token) {
        Claims claims = Jwts.parser()
                .verifyWith(key)
                .build()
                .parseSignedClaims(token)
                .getPayload();
        return new JwtPayload(
                Long.parseLong(claims.getSubject()),
                claims.get("username", String.class),
                claims.get("role", String.class));
    }

    public record JwtPayload(Long userId, String username, String role) {}
}
```

### 3.2 SecurityFilterChain 配置

```
SecurityConfig.filterChain():
  csrf.disable()
  sessionManagement.stateless()

  authorizeHttpRequests:
    /api/auth/register  → permitAll()
    /api/auth/login     → permitAll()
    /api/health         → permitAll()
    /api/**             → authenticated()

  addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter)

  authenticationManager:
    DaoAuthenticationProvider
      → CustomUserDetailsService.loadUserByUsername()
      → BCryptPasswordEncoder
```

### 3.3 JWT Filter

```java
// JwtAuthenticationFilter.java — 拦截逻辑
doFilterInternal(request, response, chain):
    String header = request.getHeader("Authorization");
    if (header == null || !header.startsWith("Bearer ")) {
        chain.doFilter(request, response);  // 交给 SecurityConfig 返回 401
        return;
    }
    try {
        String token = header.substring(7);
        JwtPayload payload = jwtUtils.validate(token);
        request.setAttribute("userId", payload.userId());
        request.setAttribute("username", payload.username());
        request.setAttribute("role", payload.role());

        UsernamePasswordAuthenticationToken auth =
            new UsernamePasswordAuthenticationToken(payload.username(), null, getAuthorities(payload.role()));
        SecurityContextHolder.getContext().setAuthentication(auth);
        chain.doFilter(request, response);
    } catch (JwtException e) {
        response.setStatus(401);
        response.setContentType("application/json;charset=UTF-8");
        response.getWriter().write("{\"code\":401,\"message\":\"Token无效或已过期\"}");
    }
```

### 3.4 Controller 获取 userId 的两种方式

```
方式一（推荐）：@RequestAttribute
  @PostMapping("/search")
  public Result search(@RequestBody SearchRequest req, @RequestAttribute("userId") Long userId) { ... }

方式二：SecurityContextHolder
  Authentication auth = SecurityContextHolder.getContext().getAuthentication();
  String username = auth.getName();  // 配合 CustomUserDetailsService
```

### 3.5 ES 索引升级策略

```
步骤1: 新建索引 localrag-chunks-v2，mapping 包含新字段
步骤2: POST _reindex { source: {index: "localrag-chunks"}, dest: {index: "localrag-chunks-v2"} }
步骤3: 旧文档的 userId/orgTag/isPublic 为 null，不影响检索（filter 会排除 null）
步骤4: POST _aliases { actions: [ {add: {index: "localrag-chunks-v2", alias: "localrag-chunks"}},
                                    {remove: {index: "localrag-chunks", alias: "localrag-chunks"}} ] }
步骤5: DELETE localrag-chunks
```

### 3.6 权限过滤的 ES bool 查询

```json
{
  "query": {
    "bool": {
      "must": [
        { "knn": { "field": "denseVector", "query_vector": [...], "k": 30 } }
      ],
      "filter": {
        "bool": {
          "should": [
            { "term": { "userId": "1" } },
            { "term": { "isPublic": true } },
            { "terms": { "orgTag": ["dev", "product"] } }
          ],
          "minimum_should_match": 1
        }
      }
    }
  }
}
```

### 3.7 Kafka payload 扩展

`FileUploadedPayload` 新增三个字段，全链路传递权限上下文：

```java
// FileUploadedPayload.java
private String md5;
private String fileName;
private long fileSize;
private String objectKey;
// 新增 ↓
private String userId;      // 从 Controller 的 @RequestAttribute 注入
private String orgTag;      // 从上传者 User.orgTag 继承
private boolean isPublic;   // 默认 false，可通过 API 修改
```

消费者（DocumentConsumer → EmbeddingConsumer → VectorStoreConsumer）在写入 ES 时将 `userId`/`orgTag`/`isPublic` 写入索引文档。
