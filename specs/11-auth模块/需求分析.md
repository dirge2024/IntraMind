# 用户认证与权限检索 — 需求分析

> 对标参考：PaiSmart `SecurityConfig`、`JwtAuthenticationFilter`、`OrgTagAuthorizationFilter`、`HybridSearchService.searchWithPermission()`

## 核心业务逻辑

为系统引入用户体系与权限控制。所有 API（除注册/登录外）需 JWT 认证，检索结果按用户 ID + 组织标签 + 公开/私有三层过滤，实现多用户数据隔离。

## 关键功能模块

### 1. 用户注册与登录 (P0)

- **注册**：POST `/api/auth/register`，body: `{username, password}`
- **登录**：POST `/api/auth/login`，返回 JWT Token
- **密码**：BCrypt 加密存储
- **角色**：USER（默认）/ ADMIN（手动升级）
- **Token**：JWT，payload 含 `sub`(userId)、`role`、`exp`(7天)，密钥从 `application.yml` 读取 Base64 字符串

### 2. JWT 认证过滤器 (P0)

- **拦截范围**：除 `/api/auth/register`、`/api/auth/login` 外所有 `/api/**`
- **提取**：从 `Authorization: Bearer <token>` 头提取
- **校验**：签名有效性 + 过期时间
- **上下文传递**：通过 `RequestAttribute` 将 `userId`、`role` 注入 Controller

### 3. 权限感知检索 (P0)

- **ES 索引扩展**：`localrag-chunks` 索引新增字段：
  - `userId: keyword` — 上传者 ID
  - `orgTag: keyword` — 组织标签
  - `isPublic: boolean` — 是否公开
- **查询 filter**：`bool.should` 三条件：
  1. `term: { userId: currentUserId }` — 自己的文档
  2. `term: { isPublic: true }` — 公开文档
  3. `terms: { orgTag: userOrgTags }` — 所属组织文档
- **无需改动**：RetrievalService 接口增加 `userId`/`orgTags` 参数，不影响现有检索逻辑

### 4. 多知识库 / 组织标签 (P1)

- **组织标签表**：`organization_tag(id, name, parent_id)` 支持层级
- **文件关联**：FileMetadata 增加 `orgTag` 字段和 `isPublic` 字段
- **简化策略**：学习阶段用单级字符串标签（如 "研发部"、"产品部"），暂不实现层级树
- **用户关联**：User 增加 `orgTag` 字段，用户可属于一个组织

### 5. 对话历史引用持久化 (P1)

- **ChatHistoryMessage** 增加 `referenceMappingsJson` 字段（LONGTEXT）
- 查看历史对话时能点击引用编号跳转源文件

## 边界条件

| 场景 | 处理策略 |
|------|----------|
| Token 过期 | 返回 401 + `{"message":"Token已过期，请重新登录"}` |
| Token 被篡改 | JWT 签名校验失败 → 401 |
| 用户名重复 | 注册时唯一索引冲突 → 400 "用户名已被占用" |
| 用户无权限访问文档 | ES filter 层隔离，不返回无权限 chunk |
| 用户属于多个组织 | orgTags 为数组，ES `terms` 查询多值 |
| 公开文档 | `isPublic=true` 对所有认证用户可见 |
| 未登录访问公开 API | `/api/auth/register`、`/api/auth/login`、`/api/health` 放行 |
| 文件上传时 userId 未写入 | Kafka 消息增加 userId 字段，消费者写入 ES 时携带 |
| 文件上传时 orgTag 未指定 | 默认继承上传者的 orgTag |

## 潜在技术难点

**1. ES 索引字段新增与 reindex**
- 难点：`localrag-chunks` 已有数据，新增 userId/orgTag/isPublic 字段需重建索引
- 方案：新建索引 `localrag-chunks-v2`（含新 mapping）→ reindex API 迁移 → 别名切换

**2. Kafka 消息携带权限上下文**
- 难点：文件上传→Kafka→消费者→ES 索引全链路异步，`document.uploaded` 消息目前无 userId
- 方案：`FileUploadedPayload` 增加 `userId`、`orgTag`、`isPublic` 字段

**3. JWT 密钥管理**
- 难点：密钥泄露可伪造 Token
- 方案：`application.yml` 读环境变量 `JWT_SECRET_KEY`，必须 Base64 编码的 256-bit+ 密钥

**4. 检索性能影响**
- 难点：ES filter 条件增加后查询变慢
- 方案：userId/orgTag 建 `keyword` 索引 + filter 走 bitmap cache，对检索速度影响极小

## API 设计

### Register
```
POST /api/auth/register
Body: {"username": "zhangsan", "password": "Abc123456"}
Response: {"code":200, "data":{"token":"eyJhbG..."}}
Error:    {"code":400, "message":"用户名已被占用"}
```

### Login
```
POST /api/auth/login
Body: {"username": "zhangsan", "password": "Abc123456"}
Response: {"code":200, "data":{"token":"eyJhbG..."}}
Error:    {"code":401, "message":"用户名或密码错误"}
```

### Search (增强)
```
POST /api/retrieval/search
Header: Authorization: Bearer <token>
Body: {"query": "GE超声校准", "topK": 5}

Response: {"code":200, "data":[
  {"chunkId":1, "md5":"...", "text":"...", "score":0.92, "fileName":"设备手册.pdf", "pageNumber":5}
]}
```

## 数据结构

### MySQL: user

```sql
CREATE TABLE user (
    id         BIGINT AUTO_INCREMENT PRIMARY KEY,
    username   VARCHAR(50)  NOT NULL UNIQUE,
    password   VARCHAR(200) NOT NULL,  -- BCrypt hash
    role       VARCHAR(20)  NOT NULL DEFAULT 'USER',  -- USER / ADMIN
    org_tag    VARCHAR(100),
    created_at DATETIME NOT NULL
);
```

### MySQL: organization_tag (P1)

```sql
CREATE TABLE organization_tag (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    parent_id   INT DEFAULT NULL,
    created_at  DATETIME NOT NULL
);
```

### ES 索引 mapping 扩展

```json
{
  "properties": {
    "userId":   { "type": "keyword" },
    "orgTag":   { "type": "keyword" },
    "isPublic": { "type": "boolean" }
  }
}
```

### FileUploadedPayload 扩展

```java
// 新增字段
String userId;
String orgTag;
boolean isPublic;
```

## Auth 模块位置

Auth 不独立为模块，合入 `api` 模块（简单场景）：

```
api/
├── config/
│   ├── SecurityConfig.java        // Spring Security + 白名单
│   └── JwtAuthenticationFilter.java
├── controller/
│   └── AuthController.java        // /register + /login
├── utils/
│   └── JwtUtils.java              // 生成 + 验证 Token
└── model/
    └── User.java                  // JPA Entity
```

**依赖**：`spring-boot-starter-security`、`jjwt-api/jjwt-impl/jjwt-jackson`、`spring-boot-starter-data-jpa`
