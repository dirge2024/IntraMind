# 框架初始化 — 技术实现方案

## 1. 基础设施（本地已有实例，不用 Docker）

### Elasticsearch
- 版本：8.10.0
- 安装路径：D:\Elasticsearch\elasticsearch-8.10.0\bin
- 默认端点：http://localhost:9200（待确认）
- 认证：已开启，用户名 `elastic`，密码 `wsy2642@`
- 开发阶段关闭 HTTPS 校验（本地自签名证书）

> ES 8.10.0 用官方 `elasticsearch-java` 客户端，版本号锁死 8.10.0，不用 Spring Data ES。

### MinIO
- 端点：http://localhost:9000
- API 端口：9000，Console 端口：9001
- Access Key：`dirge`，Secret Key：`wsy2642@`
- 开发阶段自动创建 bucket：`localrag-documents`

### Kafka
- 状态：其他项目已有实例，暂不重复部署
- 关键问题：**一个 Kafka 能否多项目共用？**

> **结论：可以。** Kafka 天然支持多租户，只需本项目的 Topic 使用独立前缀，如 `localrag.doc.upload`、`localrag.doc.processed`，与其他项目的 Topic 隔离开即可。消费者组 ID 也需独立命名。后续 messaging 模块再做对接。

### docker-compose 策略
本项目 docker-compose.yml 仅作**备选方案**，用于其他开发者在无本地中间件时快速拉起。当前开发不使用 Docker 部署 ES/MinIO/Kafka。

## 2. 父 POM

```xml
<!-- 关键版本锁定 -->
<java.version>17</java.version>
<spring-boot.version>3.3.5</spring-boot.version>
<elasticsearch.version>8.10.0</elasticsearch.version>   <!-- 与本地ES一致 -->
<minio.version>8.5.7</minio.version>
<kafka.version>3.6.0</kafka.version>                     <!-- 预留 -->
<okhttp.version>4.12.0</okhttp.version>                  <!-- ES客户端依赖 -->
```

注意事项：
- ES 客户端版本必须和本地 ES 8.10.0 精确匹配，大版本 API 不兼容
- MinIO 客户端 8.5.x 对应 Java 17，接口稳定
- 排除 spring-boot-starter-data-elasticsearch（它绑了老版 High Level Client）

## 3. common 模块

- Result<T>：code(int), message(String), data(T)，静态工厂 ok() / fail()
- BaseException：extends RuntimeException，携带 code
- GlobalExceptionHandler：@RestControllerAdvice，捕获 BaseException 和兜底 Exception
- 依赖：仅 lombok + slf4j

- api 模块额外含主启动类

## 7. ES HTTPS 证书问题

ES 8.x 默认启用 HTTPS + 自签名证书，开发环境两种处理方式：
1. 启动时关闭：`xpack.security.enabled=false`（需改本地 ES 配置并重启）
2. 跳过证书校验：配置 OkHttp SSLContext 信任所有证书（代码层处理）

> 建议方案 2，不动本地 ES 配置。在 vector-store 模块创建 ES 客户端时，用自定义 SSLContext 跳过证书校验。

## 8. Kafka 多项目共用说明

单 Kafka 实例多项目共用完全可行，隔离策略：

| 隔离层面 | 做法 | 本项示例 |
|----------|------|----------|
| Topic 命名 | 项目前缀 | `localrag.doc.upload`、`localrag.doc.vectorized` |
| 消费者组 | 独立 group-id | `localrag-doc-processor`、`localrag-embedding-worker` |
| 消息体 | 项目内定义 | JSON 序列化，含 messageId 追踪 |

只要 Topic 名前缀不冲突，多个项目互不影响，无需额外部署 Kafka 实例。
