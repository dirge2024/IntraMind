## 3. 实现策略

### 3.1 Redis 固定窗口限流

```java
// RateLimitService.java
public void checkChatByUser(String userId) {
    long minuteSlot = System.currentTimeMillis() / 60000;
    String key = "chat:rate:user:" + userId + ":" + minuteSlot;

    Long count = redisTemplate.opsForValue().increment(key);
    if (count == 1) {
        redisTemplate.expire(key, Duration.ofSeconds(60));
    }

    if (count != null && count > props.getChatPerMinutePerUser()) {
        long retryAfter = 60 - (System.currentTimeMillis() / 1000) % 60;
        throw new RateLimitExceededException("请求过于频繁，请" + retryAfter + "秒后重试",
                "user", (int) retryAfter);
    }
}
```

**窗口选择**：固定窗口（分钟取整），不引入滑动窗口复杂度。边界场景（跨分钟）用 INCR 原子操作 + TTL 兜底。

### 3.2 Token 预占与结算

```java
// UsageQuotaService.java
public TokenReservation reserveLlmUsage(String userId, int estimatedPrompt, int maxCompletion) {
    int estimated = estimatedPrompt + maxCompletion;
    String today = LocalDate.now().toString();
    String userKey = "llm:usage:user:" + userId + ":" + today;
    String globalKey = "llm:usage:global:" + today;

    Long userTotal = redisTemplate.opsForValue().increment(userKey, estimated);
    Long globalTotal = redisTemplate.opsForValue().increment(globalKey, estimated);

    // 首次设置 TTL 到次日零点
    long secondsToMidnight = ChronoUnit.SECONDS.between(
            LocalDateTime.now(), LocalDate.now().plusDays(1).atStartOfDay());

    if (userTotal != null && userTotal - estimated < props.getLlmDailyTokensPerUser()) {
        redisTemplate.expire(userKey, Duration.ofSeconds(secondsToMidnight));
    }
    if (globalTotal != null && globalTotal - estimated < props.getLlmDailyTokensGlobal()) {
        redisTemplate.expire(globalKey, Duration.ofSeconds(secondsToMidnight));
    }

    if (userTotal != null && userTotal > props.getLlmDailyTokensPerUser()) {
        redisTemplate.opsForValue().decrement(userKey, estimated);
        redisTemplate.opsForValue().decrement(globalKey, estimated);
        throw new RuntimeException("今日 Token 额度已用完");
    }

    return new TokenReservation(userId, estimated, maxCompletion, System.currentTimeMillis());
}

public void settleReservation(TokenReservation r, int actualTokens) {
    String today = LocalDate.now().toString();
    int diff = r.estimated() - actualTokens;
    if (diff > 0) {
        redisTemplate.opsForValue().decrement("llm:usage:user:" + r.userId() + ":" + today, diff);
        redisTemplate.opsForValue().decrement("llm:usage:global:" + today, diff);
    }
}
```

### 3.3 Token 估算算法

```
estimateTextTokens:
  ASCII 字符: len * 0.3
  CJK 字符:   len * 0.95
  → 总 = ceil(ASCII估算 + CJK估算)

estimateChatTokens(messages):
  每条: 8 (format overhead) + estimateTextTokens(role) + estimateTextTokens(content)
```

### 3.4 LLM Provider 抽象

```java
// LlmProvider.java — 统一流式接口
public interface LlmProvider {

    record StreamHandle(Disposable subscription, Runnable onCancel) {
        public void cancel() {
            if (subscription != null && !subscription.isDisposed()) {
                subscription.dispose();
            }
            if (onCancel != null) onCancel.run();
        }
    }

    StreamHandle streamChat(String userId,
                            List<Map<String, Object>> messages,
                            Consumer<String> onChunk,
                            Consumer<Throwable> onError,
                            Runnable onComplete);

    String getName();

    static LlmProvider deepseek(LlmConfig config, WebClient.Builder builder) { ... }
    static LlmProvider qwen(LlmConfig config, WebClient.Builder builder) { ... }
}
```

**切换策略**：`application.yml` 中 `localrag.llm.provider=deepseek/qwen`，`@ConditionalOnProperty` 选择 Bean。

### 3.6 线程池隔离

```java
@Bean("chatExecutor")
public ThreadPoolTaskExecutor chatExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    executor.setCorePoolSize(4);
    executor.setMaxPoolSize(8);
    executor.setQueueCapacity(50);
    executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
    executor.setThreadNamePrefix("chat-");
    executor.initialize();
    return executor;
}
```

- Tomcat 线程：接收 HTTP → 提交 ReActLoop 任务到 chatExecutor → 立即释放
- chatExecutor 线程：执行 ReAct 循环 → WebClient 异步 IO → 写 SSE
- 注意：SSE Emitter 必须在 Tomcat 线程上返回，内部异步写由 chatExecutor 完成

### 3.8 异常兜底

**检索降级**（`RetrievalServiceImpl.java`）：

```java
public List<RetrievalResult> search(String query, int topK, String userId, List<String> orgTags) {
    try {
        float[] queryVector = embeddingService.embed(List.of(query)).get(0);
        return hybridSearch(query, queryVector, topK, userId, orgTags);  // KNN + BM25
    } catch (Exception e) {
        log.warn("embedding failed, fallback to BM25-only: {}", e.getMessage());
        return textOnlySearch(query, topK, userId, orgTags);  // 纯 BM25
    }
}
```

**LLM 降级**（`ReActLoop` 中 `streamReActTurn`）：

```
LLM 超时(120s)      → 返回已生成部分 + "生成超时，请重试"
LLM API 返回 429    → 等待 retry-after 秒 → 重试 1 次 → 仍失败返回"服务繁忙"
LLM API 返回 5xx    → 重试 1 次 → 仍失败返回"AI 服务暂不可用"
用户取消            → dispose subscription + 返回 STOP event
```

**文件处理降级**：

```
Tika 解析失败 → FileMetadata.status = PARSE_FAILED，重试 API
向量化失败   → FileMetadata.addVectorizationStatus("FAILED")，重索引 API
```

## 4. API 路由

无新增 API 端点；限流异常通过 `GlobalExceptionHandler` 统一处理：

```
所有 /api/** 请求 → 限流检查（非拦截器，业务层调 checkChatByUser）

RateLimitExceededException 捕获:
HTTP 429 Too Many Requests
Body: {"code":429,"message":"请求过于频繁，请30秒后重试","data":{"retryAfterSeconds":30}}
```
