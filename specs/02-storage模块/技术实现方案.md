# Storage 模块 — 技术实现方案

## 1. 依赖
- `io.minio:minio:8.5.7`
- `org.springframework.boot:spring-boot-starter-web`
- `com.localrag:common`

## 2. 接口设计

```
com.localrag.storage.interface
├── StorageService              # MinIO 操作接口
│   ├── initMultipartUpload(bucket, objectKey) → uploadId
│   ├── uploadPart(bucket, objectKey, uploadId, partNumber, stream, size) → etag
│   ├── completeMultipartUpload(bucket, objectKey, uploadId, parts[]) → etag
│   ├── abortMultipartUpload(bucket, objectKey, uploadId)
│   ├── putObject(bucket, objectKey, stream, size, contentType) → etag
│   ├── getPresignedUrl(bucket, objectKey, expirySeconds) → url
│   ├── removeObject(bucket, objectKey)
│   └── bucketExists(bucket) / createBucket(bucket)
│
├── FileMetadataRepository      # 文件元数据存取
│   ├── findByMd5(md5) → FileMetadata
│   ├── save(FileMetadata)
│   ├── findByStatus(status) → List<FileMetadata>
│   └── delete(md5)
│
└── UploadStateManager          # 上传状态追踪
    ├── createTask(fileMd5, fileName, fileSize, uploadId, totalParts) → taskId
    ├── savePartEtag(taskId, partNumber, etag)
    ├── getTask(taskId) → UploadTask
    ├── getSortedParts(taskId) → List<Part>
    ├── removeTask(taskId)
    └── cleanExpiredTasks(maxAgeHours)
```

## 3. 实现策略

### MinIO 客户端初始化
```java
@Bean
public MinioClient minioClient() {
    return MinioClient.builder()
        .endpoint(config.getEndpoint())
        .credentials(config.getAccessKey(), config.getSecretKey())
        .build();
}
```
启动后自动检测 bucket，不存在则创建。

### Multipart 上传流程
1. Init: `createMultipartUpload` → 返回 `uploadId`
2. Part: `uploadPart` → 返回 `etag`（带引号的字符串，如 `"abc123"`）
3. Complete: 构建 `Part[]`，逐个 `new Part(partNumber, etag)`，调 `completeMultipartUpload`

### MD5 去重逻辑
```
initUpload(fileMd5) {
    metadata = fileMetadataRepo.findByMd5(fileMd5)
    if (metadata != null && metadata.status == READY) {
        return 秒传响应
    }
    uploadId = minio.createMultipartUpload(bucket, fileMd5 + "/" + fileName)
    taskId = uploadStateManager.createTask(...)
    return 正常响应
}
```

### 分片大小计算
```java
long minPartSize = 5 * 1024 * 1024;   // 5MB
long maxParts = 10000;
long partSize = Math.max(minPartSize, (fileSize + maxParts - 1) / maxParts);
```

### 上传状态存储
- `ConcurrentHashMap<String, UploadTask>`，线程安全
- 定时任务（`@Scheduled`）：每小时清理超过 24 小时的过期待完成上传
- 清理时顺带调 MinIO `abortMultipartUpload` 释放 MinIO 侧分片

### 文件元数据存储
- 当前：`ConcurrentHashMap<String, FileMetadata>`
- key = fileMd5
- 预留接口，后续无缝切换 DB

## 4. API 路由

```
POST   /api/storage/upload/init       → StorageController.initUpload
POST   /api/storage/upload/part       → StorageController.uploadPart
POST   /api/storage/upload/complete   → StorageController.completeUpload
DELETE /api/storage/upload/{taskId}   → StorageController.abortUpload
GET    /api/storage/upload/{taskId}   → StorageController.getProgress
POST   /api/storage/upload/direct     → StorageController.directUpload
GET    /api/storage/download/{fileId} → StorageController.download
DELETE /api/storage/file/{fileId}     → StorageController.deleteFile
```

## 5. 配置项

```yaml
localrag:
  minio:
    endpoint: ${MINIO_ENDPOINT:http://localhost:9000}
    access-key: ${MINIO_ACCESS_KEY}
    secret-key: ${MINIO_SECRET_KEY}
    bucket: ${MINIO_BUCKET:localrag-documents}
    presigned-expiry-seconds: 600
    upload-expire-hours: 24

spring:
  servlet:
    multipart:
      max-file-size: 50MB
      max-request-size: 50MB
```
